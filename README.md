# WorkInsight

## 概要
スケジュール管理、タスク管理を行い、
作業時間の中でどれだけ集中できたかを可視化する作業進捗管理アプリです

## アプリURL
https://work-insight.vercel.app
※Render無料プランのため初回アクセス時は数十秒かかる場合があります

## 開発に至った経緯
以前、家計簿アプリを個人開発した際、自分で開発計画を練った上で開発を行なっていたが、予定していた開発計画より完成が遅れてしまいました。
原因を振り返ると開発工程でモチベーションの低下や集中力の低下で、実際の作業時間と計画していた作業時間に差が生まれていることに気づきました
そこで「作業時間や集中できていない時間を可視化できれば、自分の作業効率を客観的に分析することができ、モチベーションの継続につながるのではないか」と考え、今回のWorkInsightを開発しました。

## 機能一覧
### スケジュール管理
- カレンダーの表示(FullCalendar)
- 予定の追加
- 終日予定の設定

### TODOリスト
- タスクの追加、更新、削除
- 期限、優先度の設定
- 完了したタスクの表示、非表示
### ユーザ設定
- サボり時間の自動算出
- タイマー機能
- タイマーが０になった際、フィードバックフォームで進捗管理
- 稼働時間、稼働曜日の設定
### 認証
- JWT認証
- 新規登録機能、ログイン機能

## 工夫した点
- ログイン認証
JWT認証を導入し、ユーザー認証をトークンベースで管理するようにしました。
前回作成した家計簿アプリではユーザーIDをLocalStorageに保存していましたが、
今回はSpring SecurityとJWTを使用することでより実践的な認証方式に改善しました。
- サボり認定方法
設定したタイマーが０になったらフォームが開くようになっており、フィードバックを入力できる。そのフィードバック
を入力したら稼働時間として合計され、空で送信するとサボり時間としてタイマーの時間分加算されていく
- UI
家計簿アプリでは色をたくさん使ったが今回はスケジュール管理も兼ねてることもありシンプルなUIを意識して作成した
- 初回ログイン時
初回ログイン時にユーザ設定画面に遷移するように処理

## 反省点・課題
今回初めてTailwind CSSを使用したが見やすいコードを書くのに苦労した。またロジックも難しかったのでデータサイエンス力をもっと
つけていかないといけないと身に沁みて感じた。また認証もtokenをLocalStorageで保存してあるのでこれをHttpOnlyにできるようにしていきたい
## 今後実装したい機能
家計簿アプリでWebSocketを使用したのでその知識を活かし、リアルタイムでフィードバックを共有できる掲示板を作成しても面白いと思った
またスケジュールやタスクを他ユーザと共有できる機能があっても使いやすいのではないかと思った。
